### 1-1. 소비자 관점 감사 후 개편 (2026-07-06, 되돌리지 말 것)

- **"1위" 표현 규칙**: 근거 = **김과외 사탐 "수강생 수" 기준, 현재도 1위**(사용자 확인). 그래서 카피는 반드시 **"김과외 사탐 수강생 수 1위"**로 한정 표기. "전국 1위"·"사회문화 1위"·"생활과윤리 1위" 같은 **무한정 최상급으로 되돌리지 말 것**(표시광고법 리스크). 노출도 페이지당 1~2회로 억제(과거 6~8회 도배 → 역효과). RESULTS stat sub = "김과외 앱 기준 · 현재".

- **용어**: 홈페이지도 "사설" → **"심화 모의고사"**로 통일 완료. "무한/폭발적/주 7일·24시간" 등 과장어 제거("반복", "당일 안에 답")·가짜 비율 막대(.bar/.barlbl) 삭제.

## 2. 노션 운영 시스템 (이미 완성됨 — 건드릴 땐 이 규칙 준수)

- **"💳 결제·정산 내역" DB**: 모든 결제 기록. 기본 뷰 = **"결제 기록 (언제·얼마)"**(학생·결제 날짜·금액·결제 방식·구분·메모, 날짜 내림차순).
  - **결제는 "언제 얼마" 최소 기록만** (2026-07-03 사용자 방침): 학생 + 결제 날짜 + 금액 + 결제 방식 + 구분. 가격: 1과목 50만 / 두 과목 80만.
  - **회분·자동 잔여 계산에 의존하지 않는다.** 교육비(수강권 소진)는 **사용자가 결제 날짜 ↔ 출석부를 직접 수동 대조**해서 판단.
    → Claude가 "돈 낼 사람" 자동 판정을 밀지 말 것. 회분 필드는 안 채워도 됨.
  - 과외/불꽃캠프는 '구분' 필드로 분리. 카드/계좌이체 필터 뷰·달력 뷰 있음.
  - **손익 월 매칭 (2026-07-11, 방침 확정 07-12)**: 결제 DB '손익 월'은 "📊 월별 손익(매출·비용·순이익)" DB로 가는 **관계** — 수식 자동화 불가(노션 한계). **규칙: 손익 월 = 결제 날짜와 무조건 같은 달(현금주의, 사용자 확정)**. 사용자는 결제 날짜·금액만 적고, Claude가 "손익 정리해줘" 요청 때 빈/어긋난 '손익 월'을 결제 날짜의 달로 연결·교정한다. **예약(스케줄) 자동화는 만들지 않기로 함(사용자 거부 2026-07-12) — 온디맨드만.** 월 행은 2026-04~12 존재, 부족하면 생성. 보조: 결제 DB에 '결제 월' 수식 열(날짜에서 자동 계산)과 "📅 월별 수입 (자동)" 뷰(손익 월 그룹) 있음.
  - **결제↔학생 연결 (2026-07-11)**: 결제 DB에 '불꽃캠프 학생'·'과외 학생' 관계 추가(구분이 불꽃캠프/온라인→불캠 DB, 과외/비대면→과외 DB). 양쪽 학생 DB에 '💳 결제 기록'(양방향)+'마지막 결제일'(latest_date 롤업)+'누적 결제액'(sum 롤업)+"💰 납부 현황" 뷰 생성. 백필 48/51건 완료. **빈 학생 연결도 손익 정리 때 이름 기준으로 같이 자동 매칭**. "이번 달 내야 하는지" 자동 판정은 만들지 않음(수동 대조 원칙 유지). **이름 확인 완료(2026-07-12)**: 김선아·김서진의 과외 결제는 과외 DB "한대부고 그룹수업" 행으로 연결(불캠 연결 해제). 김규림=**신규림**이 맞음(결제 행 제목 정정). 과외 학생 행 류태환→**류태완**으로 정정(환이 오타). 안지민 과외 DB에 추가. 미결: 김지연(누군지모름)·서동하(과외 DB에 행 없음).

    '토 …'면 그 주 **토요일 날짜**, '일 …'면 **일요일 날짜**로 넣는다. (주 1회 = 학생당 주 1행. 두 과목=같은 요일이라 1행.)
  - '학생'(→불꽃캠프 학생) 관계 필수.
  - 수업 날 "✅ 회차별 체크" 뷰(날짜 그룹 = 토/일 자동 분리)에서 온 학생만 **수동 체크** → 출석 횟수 자동 반영.
    미체크 행은 집계 제외.
  - **"언제 출석했는지"** = 학생 DB의 **'출석한 날들'** 롤업(대시보드 열)에 체크된 날짜만 뜸(예: "5/23, 5/30, 6/6…").
    구현: 출석부 '출석일자' 수식 = `if(prop("출석"), formatDate(prop("날짜"),"M/D"), "")` → 학생 DB '출석한 날들' 롤업(출석기록→출석일자, show_unique).
    체크 안 된 미래 주차 행은 "" 반환 → 목록에 안 섞임. (2026-07-03 구축·검증)
