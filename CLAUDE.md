## 2. 노션 운영 시스템 (이미 완성됨 — 건드릴 땐 이 규칙 준수)

- **"💳 결제·정산 내역" DB**: 모든 결제 통합 관리. 카드/현금/계좌이체 필터 뷰 있음.
  - 결제 1건 = 4주 1사이클 = 회분 4. 가격: 1과목 50만 원, 두 과목 80만 원.
  - 과외와 불꽃캠프는 결제의 **'구분' 필드**로 분리. 과외→불캠 전환율 대시보드 있음.
- **"불꽃캠프 원비" DB**: 학생별 수강권 잔액 관리.
  - 잔여회수 = 충전회수(결제 회분 합) − 누적회(출석 수).
  - 충전(결제 입력)은 사용자가 직접, 차감(출석 → '누적회' 갱신)은 Claude가 자동으로 수행.
- **상담 신청**: 홈페이지 → 노션 폼(과외 리드 트래커 DB의 폼 뷰)이 **직접 기록**.
  Make/Formspree 등 외부 자동연동은 전부 폐기됨 — 다시 제안하지 말 것.
- **워크스페이스 구조** (2026-07-02 재정리): 최상위 "🔥 불꽃캠프-과외" 아래 5개가 전부.
  1. 🎛️ 대시보드 — **리스트 딱 3개** (2026-07-03 최종 확정, 사용자 지시 "이것만"):
     ① 상담 리스트(리드 트래커 표 — 상태 열로 퍼널 관리) ② 과외 리스트(납부여부·금액·결제수단·불캠 전환 체크)
     ③ 불꽃 리스트(원비 DB — 결제 방식 롤업(카드/계좌), 잔여회수. **학생 클릭 → '시험 기록'으로 점수 연결**)
     — 헤더 바로가기 1줄: 출석부·시험·결제·인스타·매뉴얼·DB 원본
     — **더 추가하지 말 것**: 점수 테이블·액션 큐·카톡방·조교HR 등은 대시보드에 두지 않는다 (각 DB로 직접).
       점수 요주의·돈낼사람 등 필터 뷰는 각 원본 DB(점수 마스터·원비)의 뷰 탭에 있음.
     — 대시보드에 설명 텍스트 넣지 말 것 (헤더 1줄이 전부)
  2. 📸 인스타 운영 (조교) — 가이드 + 콘텐츠 보드 DB
  3. 📚 자료실 — 읽는 문서 전부. **새 문서는 무조건 여기에 생성**
  4. 🗃️ DB 원본 — 모든 DB 원본 (결제·정산, 원비, 학생, 출석부, 리드 트래커, 점수 마스터, 시험,
     💬 카톡방 운영(공지/정오표/주간 자료), 🧑‍🏫 조교 HR(채용·급여) 등)
  5. 🗄️ 아카이브 — 옛 메모·폐기 시스템. 참고 금지, 삭제도 금지
  - 새 최상위 페이지·새 카테고리를 만들지 말 것. 이 5개 구조 유지.

## 2-0. 시험·점수 시스템 (2026-07-02 구축)

- **원칙**: 시험지 파일 = 드라이브(링크만 노션에), 점수 데이터 = 노션에만. 드라이브 점수 시트("2026 불꽃사탐" 등)는 과거 기록 보관용 동결 — 점수 이중 입력 금지.
- **이관 완료 (2026-07-02)**: "2027 학생 시트"의 4/18~6/28 점수 **518건**(현재 점수 마스터 = 이 518건이 전부, 응시일 2026-04-18~06-28) + 시험 64개를 노션으로 이관. 원비 미등록이라 학생 연결 없이 들어간 이름: 신우진·유진모·한준석·김지후·이수민·왕서준·김서진·준우·홍예준·윤원준·정서윤·신규림 (정서윤↔장서윤, 신규림↔김규림은 오타 의심 — 사용자 확인 필요). "2026 불꽃사탐" 시트는 작년(2026학년도) 기록이라 이관하지 않음.
- **작년·샘플 12건 제거 (2026-07-03)**: 이관 전부터 점수 마스터에 있던 작년/샘플 12행(이유찬·한규연·김민서 등, url 38516d93…, 응시일·시험 연결 없음)을 🗄️ 아카이브로 이동. 점수 마스터에는 올해 데이터만 남김. (하드 삭제는 안 함 — 아카이브에서 최종 삭제 가능)
- **"🧾 시험" DB**: 시험 1개 = 1행 (시험명·과목·유형(OX/기출/사설)·시험일·시험지 드라이브 링크).
  - '기반 기출' 자기참조 관계: OX 모의고사는 이전 주 기출 2개로 제작 (3중 복습 2차).
  - 주간 구성 고정: 사문 = OX 1 + 기출 2 + 사설 2 / 생윤 = OX 2 + 기출 4.
- **"🎯 점수 마스터" DB**: 학생×시험 = 1행. **행 제목(title '학생')은 "이름 · 시험명" 형식**(예: "차현서 · 사문 23-3") — 학생 페이지의 '시험 기록' 목록이 읽히도록. 새 점수 입력 시에도 이 형식 유지. '학생 연결'(→원비 DB, 양방향 '시험 기록') + '시험 연결'(→시험 DB) + 1·2·3회차/최종. 상승폭·만점도달 자동 수식.
  - 원비 DB에서 학생 페이지를 열면 '시험 기록'으로 시험별 재시험 점수가 보임.
  - 대시보드 ⑥ 점수 뷰는 '응시일' 그룹(날짜별), 점수 마스터 DB 자체의 '학생별 추이'·'시험별 통계' 뷰는 관계 그룹.
  - 기존 12행(이유찬·한규연 등)은 현행 학생 명단과 이름 불일치 — 작년/샘플 데이터로 추정, 관계 없이 유지.
- 수업 루프: 모의고사 → 조교 1:1 피드백 → 그날 100점까지 재시험. 3중 복습: 이번 주 기출 → 다음 주 OX 재료 → 4주 후 재응시.

## 2-1. 인스타 운영 (미대생 조교 위임, 2026-07 시작)

- 목적: 팔로워 수가 아니라 "대치 사탐 = 장쌤" 각인 → 홈페이지 상담폼 유입. 모든 게시물 CTA = 프로필 링크.
- 노션 "📸 인스타 운영 (조교)" 페이지에 가이드 5개 항목 + "📸 인스타 콘텐츠 보드" DB(상태: 아이디어→초안→장쌤 확인 필요→게시 완료).
- 불변 규칙: 학생 신상 동의 없이 노출 금지 / 성적 보장 표현 금지 / 타 강사·업체 언급 금지 / 숫자(성적·데이터) 포함 콘텐츠만 장쌤 검수 필수. 나머지는 조교 재량.

## 3. 출석 시스템 (2026-07-03 전환: 체크 기반 자동 집계 — 드라이브 파싱 루틴 폐기)

- **현행 흐름**: 수업 날 노션 **"🗓️ 출석부" DB**에 학생당 한 줄 추가('원비 학생' 관계 연결 + 출석 체크)
  → 원비 DB의 **'누적회(자동)'**(출석부 체크 rollup)가 실시간 집계
  → **잔여회수 수식 = 충전회수 − 누적회(자동)** → '상태'가 "🔴 결제 필요"로 바뀌면 대시보드 돈낼사람 큐에 뜸.
- 4/18~6/28 출석 150건은 시트에서 백필 완료 (원비 학생별 합계가 기존 누적회와 전원 일치 검증됨).
- 원비의 **'누적회'(수동 숫자 필드)는 레거시** — 더 이상 갱신하지 않음. 삭제 금지(과거 대조용).
- **드라이브 "2027 학생 시트" 파싱 루틴은 폐기.** "출석 갱신해줘" 요청이 오면 이 체크 기반 시스템을 안내할 것.
  점수→출석 유추도 금지 (오류 원인이었음). 출석 = 출석부 체크가 유일한 원천.
- **회차별 출석부 운영**: 주차마다 활성(그만둠 아님) 원비 학생 전원의 행을 **미리 생성**(토요일 날짜, 미체크,
  '원비 학생' 관계 필수). 수업 날 출석부 "✅ 회차별 체크" 뷰에서 온 학생만 체크 → 잔여 자동 차감.
  미체크 행은 집계에 안 잡히므로 잔여에 영향 없음. 주 1회 카운트 규칙 = 학생당 주 1행.
  (7/4 주차 16명 생성됨. 새 학생이 원비에 추가되면 다음 주차 생성 때 자동 포함;
   당주에 바로 넣으려면 사용자가 요청 시 해당 학생 행 1개 추가.)
- **주차 갱신** (사용자가 "주차 갱신/넘겨줘" 하면 두 가지 실행):
  1. 출석부에 새 주차 행 생성 — 그 시점의 활성 원비 학생 전원 (위 규칙).
  2. 점수 마스터 DB의 "🔴 요주의 (이번 주차)" 뷰(view://39216d93-cc5f-816c-ad6e-000ccfc3093e)의
     응시일 필터 경계를 이번 주차 토요일로 갱신
     (`CLEAR FILTER; FILTER "응시일" >= "<토요일 날짜>"; FILTER "최종" < 45 OR "1회차" < 30`).
     상대 날짜("past week")는 DSL이 저장해도 작동 안 함 — 명시적 날짜만.
