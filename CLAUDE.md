## 3. 출석 시스템 (2026-07-03 전환: 체크 기반 자동 집계 — 드라이브 파싱 루틴 폐기)

- **현행 흐름**: 수업 날 노션 **"🗓️ 출석부" DB**에 학생당 한 줄 추가 + 출석 체크. 두 관계를 **모두** 연결해야 함:
  '학생'(→👤 불꽃캠프 학생) + '원비 학생'(→원비 DB). 둘 다 걸어야 아래 두 집계가 동시에 맞음.
  → 👤 불꽃캠프 학생의 **'출석 횟수'**(rollup, 대시보드에 표시) + 원비의 **'누적회(자동)'**(rollup) 실시간 집계
  → **잔여회수 = 충전회수 − 누적회(자동)** → '상태'="🔴 결제 필요"면 원비의 돈낼사람 뷰에 뜸.
- 4/18~6/28 출석 150건 백필 완료. 두 rollup 모두 검증됨(순사랑 8·차윤서 8·김동화 6·정의현 8·장서윤 3·박민서 4 = 기대치 일치).
  두 학생 DB 역할: **👤 불꽃캠프 학생 = 명단·반/시간·출석·시험**(대시보드 메인) / **원비 = 충전·잔여·결제**(돈낼사람). 이름으로 연결(관계 아님).
- 원비의 **'누적회'(수동 숫자 필드)는 레거시** — 더 이상 갱신하지 않음. 삭제 금지(과거 대조용).
- **드라이브 "2027 학생 시트" 파싱 루틴은 폐기.** "출석 갱신해줘" 요청이 오면 이 체크 기반 시스템을 안내할 것.

- **회차별 출석부 운영** (2026-07-03: 요일별 분리):
  - 주차마다 활성(종료·작년학생 아님) **👤 불꽃캠프 학생** 전원의 행을 미리 생성. **요일 분리**: 학생의 상태(Stage)가
    '토 …'면 그 주 **토요일 날짜**, '일 …'면 **일요일 날짜**로 넣는다. (주 1회 = 학생당 주 1행. 두 과목=같은 요일이라 1행.)
  - '학생'(→불꽃캠프 학생) 관계 필수. '원비 학생'은 원비에 같은 이름 있으면 함께 연결(잔여 차감용, 없으면 생략).
  - 수업 날 "✅ 회차별 체크" 뷰(날짜 그룹 = 토/일 자동 분리)에서 온 학생만 **수동 체크** → 출석 횟수·잔여 자동 반영.
    미체크 행은 집계 제외.
  - **"언제 출석했는지"** = 학생 DB의 **'출석한 날들'** 롤업(대시보드 열)에 체크된 날짜만 뜸(예: "5/23, 5/30, 6/6…").
    구현: 출석부 '출석일자' 수식 = `if(prop("출석"), formatDate(prop("날짜"),"M/D"), "")` → 학생 DB '출석한 날들' 롤업(출석기록→출석일자, show_unique).
    체크 안 된 미래 주차 행은 "" 반환 → 목록에 안 섞임. (2026-07-03 구축·검증)
  - 현재: 7/4(토) 13명 + 7/5(일) 6명 = 19명 생성됨. 로스터·요일은 학생 DB의 상태(Stage) 기준.
